DARIA Issue Tracker Guide

Introduction

The DARIA Issue Tracker has been enhanced to support a complete UX research-to-prototype workflow. It now allows teams to track research findings, design opportunities, epics, and user stories, and generate AI-powered prototypes directly from your research.

Research-to-Prototype Workflow

The enhanced workflow follows these steps:
  1. Insights: Capture research insights and observations in Opportunity issues
  2. Design: Break down opportunities into Epic issues that represent major features
  3. Implementation: Create detailed User Story issues for specific functionality
  4. Prototyping: Generate Cursor AI prompts from your issues to automatically create working prototypes

Issue Types

Type Description When to Use
Opportunity A high-level insight or need identified through user research When you've discovered a user need or problem space through interviews, observations, or data analysis
Epic A large feature or capability that addresses an opportunity When you need to break down a large opportunity into manageable pieces of work
User Story A specific piece of functionality from a user's perspective When you need to describe a specific interaction or capability for implementation
Bug A defect or error in existing functionality When something isn't working as expected
Feature A new capability or enhancement When you want to add new functionality outside the research workflow
Improvement An enhancement to existing functionality When you want to improve something that already exists
Task A general work item that doesn't fit other categories For general tasks, maintenance work, or documentation

Status Types

Status Description
Open Issue has been created but work hasn't started
In Design Design work is in progress (for Opportunities, Epics, User Stories)
In Progress Work is actively being done on the issue
Ready for Cursor Issue is ready to be converted to a Cursor AI prompt for prototyping
Prototyped A prototype has been generated using Cursor AI
Resolved Work is complete but awaiting verification
Closed Issue has been completed and verified
Backlog Issue has been deferred for future consideration

Key Fields

Purpose: Establishes relationships between issues in the research-to-prototype workflow.

  • Epics should have an Opportunity as their parent
  • User Stories should have an Epic as their parent

Example: If researching a "Simplified Onboarding" opportunity, you might create an "Account Creation" epic with several user stories like "Email Verification" and "Profile Setup".

Purpose: Connects issues to specific user personas from your research.

Best used with Opportunities and User Stories to maintain focus on specific user needs.

Example: "New User Nina" or "Expert User Eric" - these persona names should match your research documentation.

Purpose: Identifies where in the user journey this issue occurs.

Helps maintain context about when and where users encounter specific needs or problems.

Example stages: "Discovery", "Onboarding", "Daily Use", "Advanced Usage", "Support"

Purpose: Captures key research findings that inform the issue.

Most important for Opportunity issues, but can be relevant for all issue types to maintain the connection to research.

Example: "Users struggle with too many options during onboarding" or "Interviews revealed confusion about navigation"

Purpose: Highlights ethical considerations related to the issue.

Helps ensure responsible design that considers potential impacts on users and society.

Example: "Consider accessibility for users with disabilities" or "Ensure data collection is transparent and minimized"

Purpose: Contains a template for generating an AI prompt that will create a working prototype using Cursor AI.

Most relevant for User Stories when they reach the "Ready for Cursor" status.

Example:

Create a component for email verification that:
- Sends a verification code to the user's email
- Allows the user to enter the code
- Shows clear success/error messages
- Works on both mobile and desktop

The component should match our design system with blue primary buttons and should handle various error states gracefully.

Workflow Example

Example Research-to-Prototype Workflow
  1. User Research: Conduct interviews with users about their onboarding experience
  2. Create an Opportunity:
    • Title: "Streamline User Onboarding"
    • Type: Opportunity
    • Linked Persona: "New User Nina"
    • Journey Stage: "Onboarding"
    • Insights: "Users abandon signup when asked for too much information", "Users expect to be able to try before creating account"
    • Ethics: "Consider privacy implications of data collection during signup"
  3. Create an Epic:
    • Title: "Progressive Registration Process"
    • Type: Epic
    • Parent Issue: "Streamline User Onboarding" (the Opportunity)
    • Description: "Create a multi-step registration process that collects minimal information upfront and progressively adds more details as users engage with the product."
  4. Create User Stories:
    • Title: "Email-only Initial Signup"
    • Type: User Story
    • Parent Issue: "Progressive Registration Process" (the Epic)
    • Linked Persona: "New User Nina"
    • Description: "As a new user, I want to create an account using only my email address, so I can quickly access the system without filling out a lengthy form."
    • Cursor Prompt Template: "Create a React component for an email-only signup form that validates email format, shows appropriate loading states, and handles server errors gracefully. The component should use our design system colors (#3366CC for primary buttons) and include responsive design for mobile and desktop."
  5. Move to Ready for Cursor: When the User Story is fully defined, move it to "Ready for Cursor" status
  6. Generate Prototype: Use the Cursor Prompt Template to generate a working prototype with Cursor AI
  7. Mark as Prototyped: Once the prototype is created, update the status to "Prototyped"
